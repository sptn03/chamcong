# Database PostgreSQL Chuẩn & Tối Ưu Nhất — App Chấm Công

> Tài liệu này mô tả chi tiết schema PostgreSQL được thiết kế chuẩn chỉnh cho môi trường Production của hệ thống chấm công. Đã loại bỏ hoàn toàn các trường thừa, tối ưu hóa kiểu dữ liệu, các ràng buộc toàn vẹn dữ liệu, chỉ mục (index) đơn và tổ hợp, cùng việc tích hợp xử lý địa lý bằng **PostGIS**.

## 1. Bản Vẽ Database (SQL DDL)

```sql
-- =========================================================
-- EXTENSIONS
-- =========================================================
CREATE EXTENSION IF NOT EXISTS pgcrypto;
CREATE EXTENSION IF NOT EXISTS postgis;

-- =========================================================
-- ENUM TYPES
-- =========================================================
CREATE TYPE user_status AS ENUM ('active', 'locked');
CREATE TYPE membership_role AS ENUM ('admin', 'employee');
CREATE TYPE employee_status AS ENUM ('active', 'locked', 'resigned');
CREATE TYPE gender_type AS ENUM ('male', 'female', 'other');
CREATE TYPE device_platform AS ENUM ('ios', 'android');
CREATE TYPE device_status AS ENUM ('pending', 'approved', 'rejected', 'revoked');
CREATE TYPE login_request_type AS ENUM ('new_login', 'new_device');
CREATE TYPE approval_status AS ENUM ('pending', 'approved', 'rejected');
CREATE TYPE wifi_match_mode AS ENUM ('ssid', 'ssid_bssid');
CREATE TYPE attendance_method AS ENUM ('gps', 'wifi', 'gps_wifi');
CREATE TYPE shift_assignment_scope AS ENUM ('company', 'branch', 'department', 'employee');
CREATE TYPE punch_type AS ENUM ('in', 'out');
CREATE TYPE attendance_source AS ENUM ('online', 'offline', 'admin_edit');
CREATE TYPE work_status AS ENUM ('normal', 'late', 'early', 'late_early', 'forgot', 'absent');

-- =========================================================
-- FUNCTION & TRIGGER ĐỂ TỰ ĐỘNG CẬP NHẬT updated_at
-- =========================================================
CREATE OR REPLACE FUNCTION set_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- =========================================================
-- FUNCTION & TRIGGER ĐỂ TỰ ĐỘNG ĐỒNG BỘ GEOGRAPHY POINT (POSTGIS)
-- =========================================================
CREATE OR REPLACE FUNCTION set_geography_point()
RETURNS TRIGGER AS $$
BEGIN
  IF NEW.lat IS NOT NULL AND NEW.lng IS NOT NULL THEN
    NEW.geom = ST_SetSRID(ST_MakePoint(NEW.lng, NEW.lat), 4326)::geography;
  ELSE
    NEW.geom = NULL;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- =========================================================
-- 1. USERS (Tài khoản Đăng nhập Hệ thống)
-- =========================================================
CREATE TABLE users (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  phone VARCHAR(20) NOT NULL UNIQUE,
  email VARCHAR(190) UNIQUE,
  password_hash VARCHAR(255),
  hunonic_sub VARCHAR(190) UNIQUE, -- ID tài khoản từ liên kết Hunonic
  status user_status NOT NULL DEFAULT 'active',
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TRIGGER trg_users_updated_at BEFORE UPDATE ON users
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 2. COMPANIES (Công ty)
-- =========================================================
CREATE TABLE companies (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name VARCHAR(190) NOT NULL,
  code VARCHAR(60) NOT NULL,
  timezone VARCHAR(60) NOT NULL DEFAULT 'Asia/Ho_Chi_Minh',
  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TRIGGER trg_companies_updated_at BEFORE UPDATE ON companies
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 3. BRANCHES (Chi nhánh)
-- =========================================================
CREATE TABLE branches (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  company_id BIGINT NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  name VARCHAR(190) NOT NULL,
  address VARCHAR(255),
  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TRIGGER trg_branches_updated_at BEFORE UPDATE ON branches
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 4. DEPARTMENTS (Phòng ban thuộc Chi nhánh)
-- =========================================================
CREATE TABLE departments (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  company_id BIGINT NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  branch_id BIGINT NOT NULL REFERENCES branches(id) ON DELETE CASCADE,
  name VARCHAR(190) NOT NULL,
  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TRIGGER trg_departments_updated_at BEFORE UPDATE ON departments
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 5. EMPLOYEES (Hồ sơ Nhân viên từng Công ty)
-- =========================================================
CREATE TABLE employees (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  company_id BIGINT NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  branch_id BIGINT NOT NULL REFERENCES branches(id) ON DELETE RESTRICT,
  department_id BIGINT NOT NULL REFERENCES departments(id) ON DELETE RESTRICT,
  employee_code VARCHAR(60) NOT NULL, -- Sẽ tạo Unique Index dạng Partial
  full_name VARCHAR(190) NOT NULL,
  email VARCHAR(190),
  phone VARCHAR(20), -- Số điện thoại liên hệ cá nhân, có thể khác SĐT đăng nhập
  birthday DATE,
  gender gender_type,
  title VARCHAR(120),
  status employee_status NOT NULL DEFAULT 'active',
  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT uq_employee_user_company UNIQUE (id, user_id, company_id) -- Tổ hợp unique để ràng buộc chéo
);

CREATE TRIGGER trg_employees_updated_at BEFORE UPDATE ON employees
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 6. COMPANY MEMBERSHIPS (Quyền của User trong từng Công ty)
-- =========================================================
CREATE TABLE company_memberships (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  company_id BIGINT NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  employee_id BIGINT,
  role membership_role NOT NULL DEFAULT 'employee',
  active_department_id BIGINT REFERENCES departments(id) ON DELETE SET NULL, -- Ghi nhận phòng ban hiện tại đang làm việc
  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  -- Ràng buộc khóa ngoại tổ hợp chéo, chống ghi lệch employee_id của user khác
  CONSTRAINT fk_membership_employee_user_company FOREIGN KEY (employee_id, user_id, company_id) REFERENCES employees(id, user_id, company_id) ON DELETE CASCADE
);

CREATE TRIGGER trg_company_memberships_updated_at BEFORE UPDATE ON company_memberships
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 7. DEVICES (Thiết bị đăng ký chấm công)
-- =========================================================
CREATE TABLE devices (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  device_uid VARCHAR(190) NOT NULL,
  device_name VARCHAR(190),
  platform device_platform NOT NULL,
  os_version VARCHAR(60),
  app_version VARCHAR(60),
  push_token VARCHAR(255),
  status device_status NOT NULL DEFAULT 'pending',
  last_login_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

);

CREATE TRIGGER trg_devices_updated_at BEFORE UPDATE ON devices
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 8. TOKENS (Quản lý Phiên đăng nhập & Token xác thực)
-- =========================================================
CREATE TABLE tokens (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  device_id BIGINT REFERENCES devices(id) ON DELETE SET NULL,
  token VARCHAR(255) NOT NULL UNIQUE,
  active SMALLINT NOT NULL DEFAULT 1 CHECK (active IN (0, 1)), -- 1 là hoạt động, 0 là không hoạt động
  ip_address VARCHAR(60),
  user_agent VARCHAR(255),
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TRIGGER trg_tokens_updated_at BEFORE UPDATE ON tokens
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 9. LOGIN APPROVAL REQUESTS (Yêu cầu duyệt thiết bị/đăng nhập)
-- =========================================================
CREATE TABLE login_approval_requests (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  employee_id BIGINT REFERENCES employees(id) ON DELETE CASCADE,
  device_id BIGINT NOT NULL REFERENCES devices(id) ON DELETE CASCADE,
  request_type login_request_type NOT NULL,
  status approval_status NOT NULL DEFAULT 'pending',
  ip_address VARCHAR(60),
  user_agent VARCHAR(255),
  reviewed_by BIGINT REFERENCES users(id) ON DELETE SET NULL,
  reviewed_at TIMESTAMPTZ,
  rejection_reason VARCHAR(255),
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TRIGGER trg_login_approval_requests_updated_at BEFORE UPDATE ON login_approval_requests
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 10. LOCATIONS (Danh mục Vị trí GPS Văn phòng)
-- =========================================================
CREATE TABLE locations (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  company_id BIGINT NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  name VARCHAR(190) NOT NULL,
  address VARCHAR(255),
  lat NUMERIC(10,7) NOT NULL CHECK (lat BETWEEN -90 AND 90),
  lng NUMERIC(10,7) NOT NULL CHECK (lng BETWEEN -180 AND 180),
  radius_m INTEGER NOT NULL DEFAULT 80 CHECK (radius_m > 0),
  geom geography(Point, 4326), -- Tự động đồng bộ qua trigger
  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TRIGGER trg_locations_updated_at BEFORE UPDATE ON locations
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

CREATE TRIGGER trg_locations_geom BEFORE INSERT OR UPDATE OF lat, lng ON locations
FOR EACH ROW EXECUTE FUNCTION set_geography_point();

-- =========================================================
-- 11. LOCATION SCOPES (Phạm vi địa lý áp dụng cho Branch/Dept)
-- =========================================================
CREATE TABLE location_scopes (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  location_id BIGINT NOT NULL REFERENCES locations(id) ON DELETE CASCADE,
  branch_id BIGINT REFERENCES branches(id) ON DELETE CASCADE,
  department_id BIGINT REFERENCES departments(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT chk_location_scope_not_empty CHECK (branch_id IS NOT NULL OR department_id IS NOT NULL)
);

-- =========================================================
-- 12. LOCATION EMPLOYEE OVERRIDES (Vị trí đặc cách riêng cho nhân viên)
-- =========================================================
CREATE TABLE location_employee_overrides (

  location_id BIGINT NOT NULL REFERENCES locations(id) ON DELETE CASCADE,
  employee_id BIGINT NOT NULL REFERENCES employees(id) ON DELETE CASCADE,

  CONSTRAINT uq_location_employee UNIQUE (location_id, employee_id)
);

-- =========================================================
-- 13. WIFIS (Danh mục Wifi được phép chấm công)
-- =========================================================

  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,

  ssid VARCHAR(190) NOT NULL,

  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

CREATE TRIGGER trg_wifis_updated_at BEFORE UPDATE ON wifis

-- =========================================================
-- 14. WIFI SCOPES (Phạm vi Wifi áp dụng cho Branch/Dept)
-- =========================================================
CREATE TABLE wifi_scopes (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  wifi_id BIGINT NOT NULL REFERENCES wifis(id) ON DELETE CASCADE,
  branch_id BIGINT REFERENCES branches(id) ON DELETE CASCADE,
  department_id BIGINT REFERENCES departments(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT chk_wifi_scope_not_empty CHECK (branch_id IS NOT NULL OR department_id IS NOT NULL)

-- =========================================================
-- 15. SHIFTS (Cấu hình Ca làm)
-- =========================================================
CREATE TABLE shifts (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  company_id BIGINT NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  name VARCHAR(190) NOT NULL,
  start_time INTERVAL NOT NULL, -- Độ lệch giờ bắt đầu ca kể từ 00:00 ngày công (ví dụ: '08:00:00' hoặc '22:00:00')
  end_time INTERVAL NOT NULL,   -- Độ lệch giờ kết thúc ca kể từ 00:00 ngày công (ví dụ: '17:00:00' hoặc '30:00:00' cho ca đêm)
  checkin_from INTERVAL NOT NULL,
  checkin_to INTERVAL NOT NULL,
  checkout_from INTERVAL NOT NULL,
  checkout_to INTERVAL NOT NULL,
  weekdays TEXT[] NOT NULL,
  attendance_method attendance_method NOT NULL,
  late_threshold_min INTEGER NOT NULL DEFAULT 0 CHECK (late_threshold_min >= 0),
  early_threshold_min INTEGER NOT NULL DEFAULT 0 CHECK (early_threshold_min >= 0),
  work_credit NUMERIC(4,2) NOT NULL DEFAULT 1.00 CHECK (work_credit >= 0), -- Hệ số công trực tiếp (ví dụ: 1.00 công, 0.50 công)
  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT chk_shifts_weekdays_not_empty CHECK (array_length(weekdays, 1) > 0),
  CONSTRAINT chk_shifts_weekdays_values CHECK (weekdays <@ ARRAY['mon','tue','wed','thu','fri','sat','sun']::text[])
);

CREATE TRIGGER trg_shifts_updated_at BEFORE UPDATE ON shifts
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 16. SHIFT ASSIGNMENTS (Gán ca cho Nhân viên/Phòng ban/Chi nhánh)
-- =========================================================
CREATE TABLE shift_assignments (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  shift_id BIGINT NOT NULL REFERENCES shifts(id) ON DELETE CASCADE,
  scope_type shift_assignment_scope NOT NULL,
  company_id BIGINT NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  branch_id BIGINT REFERENCES branches(id) ON DELETE CASCADE,
  department_id BIGINT REFERENCES departments(id) ON DELETE CASCADE,
  employee_id BIGINT REFERENCES employees(id) ON DELETE CASCADE,
  starts_on DATE,
  ends_on DATE,
  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT chk_shift_assignment_scope CHECK (
    (scope_type = 'company' AND branch_id IS NULL AND department_id IS NULL AND employee_id IS NULL)
    OR
    (scope_type = 'branch' AND branch_id IS NOT NULL AND department_id IS NULL AND employee_id IS NULL)
    OR
    (scope_type = 'department' AND branch_id IS NOT NULL AND department_id IS NOT NULL AND employee_id IS NULL)
    OR
    (scope_type = 'employee' AND employee_id IS NOT NULL AND branch_id IS NULL AND department_id IS NULL)
  ),
  CONSTRAINT chk_shift_assignment_date_range CHECK (
    ends_on IS NULL OR starts_on IS NULL OR ends_on >= starts_on
  )
);

CREATE TRIGGER trg_shift_assignments_updated_at BEFORE UPDATE ON shift_assignments
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 17. ATTENDANCE RECORDS (Bảng ghi công tổng hợp theo ngày-nhân viên-ca)
-- =========================================================
CREATE TABLE attendance_records (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  company_id BIGINT NOT NULL REFERENCES companies(id) ON DELETE RESTRICT,
  employee_id BIGINT NOT NULL REFERENCES employees(id) ON DELETE RESTRICT,
  shift_id BIGINT NOT NULL REFERENCES shifts(id) ON DELETE RESTRICT,
  work_date DATE NOT NULL, -- Ngày tính công (Ngày bắt đầu ca)
  checkin_at TIMESTAMPTZ,
  checkout_at TIMESTAMPTZ,
  source attendance_source NOT NULL DEFAULT 'online',
  approval_status approval_status NOT NULL DEFAULT 'approved',
  work_status work_status NOT NULL DEFAULT 'normal',
  late_min INTEGER NOT NULL DEFAULT 0 CHECK (late_min >= 0),
  early_min INTEGER NOT NULL DEFAULT 0 CHECK (early_min >= 0),
  work_credit NUMERIC(6,2) NOT NULL DEFAULT 0.00 CHECK (work_credit >= 0),
  approved_by BIGINT REFERENCES users(id) ON DELETE SET NULL,
  approved_at TIMESTAMPTZ,
  rejection_reason VARCHAR(255),
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT uq_attendance_employee_shift_date UNIQUE (employee_id, shift_id, work_date)
);

CREATE TRIGGER trg_attendance_records_updated_at BEFORE UPDATE ON attendance_records
FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- =========================================================
-- 18. ATTENDANCE EVIDENCES (Bằng chứng chấm công chi tiết vào/ra)
-- =========================================================
CREATE TABLE attendance_evidences (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  attendance_record_id BIGINT NOT NULL REFERENCES attendance_records(id) ON DELETE CASCADE,
  punch_type punch_type NOT NULL,
  device_id BIGINT REFERENCES devices(id) ON DELETE SET NULL,
  client_time TIMESTAMPTZ NOT NULL,
  server_time TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  lat NUMERIC(10,7) CHECK (lat BETWEEN -90 AND 90),
  lng NUMERIC(10,7) CHECK (lng BETWEEN -180 AND 180),
  accuracy_m NUMERIC(8,2),
  geom geography(Point, 4326), -- Tự động đồng bộ qua trigger để chạy spatial queries
  wifi_ssid VARCHAR(190),
  wifi_bssid VARCHAR(60),
  photo_path VARCHAR(255),
  photo_hash VARCHAR(255), -- Chống thay đổi ảnh khi gửi offline
  note VARCHAR(255),
  gps_valid BOOLEAN NOT NULL DEFAULT FALSE,
  wifi_valid BOOLEAN NOT NULL DEFAULT FALSE,
  distance_m NUMERIC(10,2), -- Khoảng cách thực tế đo được
  matched_location_id BIGINT REFERENCES locations(id) ON DELETE SET NULL,
  matched_wifi_id BIGINT REFERENCES wifis(id) ON DELETE SET NULL,
  validation_error VARCHAR(255),
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT chk_evidence_content CHECK (
    (lat IS NOT NULL AND lng IS NOT NULL) OR (wifi_ssid IS NOT NULL) OR (photo_path IS NOT NULL)
  )

-- =========================================================
-- 19. ATTENDANCE EDIT LOGS (Nhật ký Admin sửa công)
-- =========================================================
CREATE TABLE attendance_edit_logs (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  attendance_record_id BIGINT NOT NULL REFERENCES attendance_records(id) ON DELETE CASCADE,
  edited_by BIGINT NOT NULL REFERENCES users(id) ON DELETE RESTRICT,
  before_json JSONB,
  after_json JSONB NOT NULL,
  reason VARCHAR(255) NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- =========================================================
-- 20. NOTIFICATIONS (Thông báo hệ thống)
-- =========================================================
CREATE TABLE notifications (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  company_id BIGINT REFERENCES companies(id) ON DELETE CASCADE,
  user_id BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  type VARCHAR(60) NOT NULL,
  title VARCHAR(190) NOT NULL,
  body TEXT,
  data_json JSONB,
  read_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- =========================================================
-- TỐI ƯU HÓA CHỈ MỤC (INDEXING) - TẬN DỤNG TỐI ĐA BỘ LỌC QUERY
-- =========================================================

-- 1. Index Toàn vẹn & Lọc Soft Delete (Partial Indexes)
CREATE UNIQUE INDEX uq_company_code_active ON companies (code) WHERE deleted_at IS NULL;
CREATE UNIQUE INDEX uq_employee_company_code_active ON employees (company_id, employee_code) WHERE deleted_at IS NULL;
CREATE UNIQUE INDEX uq_membership_user_company_active ON company_memberships (user_id, company_id) WHERE deleted_at IS NULL;

-- 2. Index cho Khoá ngoại (Tránh Full Table Scan khi thực hiện JOIN)
CREATE INDEX idx_branches_company_id ON branches(company_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_departments_branch_id ON departments(branch_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_employees_user_id ON employees(user_id) WHERE status = 'active' AND deleted_at IS NULL;
CREATE INDEX idx_company_memberships_employee_id ON company_memberships(employee_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_tokens_user_id ON tokens(user_id) WHERE active = 1 AND revoked_at IS NULL;
CREATE INDEX idx_devices_user_id ON devices(user_id);

-- 3. Composite Indexes - Tối ưu hoá truy vấn Báo cáo & Lịch sử
-- Tối ưu cho query thống kê công của phòng ban/toàn công ty theo khoảng thời gian
CREATE INDEX idx_attendance_records_company_date ON attendance_records(company_id, work_date);
-- Tối ưu cho query chi tiết công của một nhân viên cụ thể theo khoảng thời gian
CREATE INDEX idx_attendance_records_employee_date ON attendance_records(employee_id, work_date);
-- Hỗ trợ tìm kiếm nhanh các bản ghi chấm công cần được duyệt
CREATE INDEX idx_attendance_records_pending_approval ON attendance_records(company_id, approval_status) WHERE approval_status = 'pending';

-- 4. Index Không gian (Spatial Index) - Giúp tìm điểm GPS cực nhanh bằng PostGIS
CREATE INDEX idx_locations_geom ON locations USING GIST(geom) WHERE deleted_at IS NULL;
CREATE INDEX idx_attendance_evidences_geom ON attendance_evidences USING GIST(geom);
```

## 2. Giải thích thiết kế & Tính năng nổi bật của từng nhóm bảng

### Nhóm 1: Người dùng & Nhân sự (`users`, `companies`, `branches`, `departments`, `employees`, `company_memberships`)
* **`users`**: Lưu thông tin xác thực gốc (SĐT, Email, Password, OAuth Google/Apple/Hunonic). Tách biệt khỏi thông tin công việc để hỗ trợ **Multi-tenant**. Một người dùng có thể làm việc cho nhiều công ty khác nhau.
* **`companies`**, **`branches`**, **`departments`**: Cấu trúc tổ chức chuẩn phân cấp. Áp dụng `timezone` lưu trực tiếp ở cấp Công ty giúp giải quyết chính xác bài toán so sánh múi giờ địa phương khi nhân viên chấm công (mặc định `'Asia/Ho_Chi_Minh'`).
* **`employees`**: Lưu hồ sơ nhân sự cụ thể tại một công ty. Phân quyền và thông tin mã nhân viên độc lập theo từng thực thể công ty.
* **`company_memberships`**: Cầu nối quyền hạn của `user` tại `company`. Chứa cột `active_department_id` để biết phiên làm việc hiện tại nhân viên đang hoạt động ở phòng ban nào.
* **Tối ưu hóa**: 
  * Sử dụng **`deleted_at TIMESTAMPTZ`** (xóa mềm) cho tất cả các bảng nhân sự quan trọng nhằm bảo toàn dữ liệu lịch sử công.
  * Sử dụng **Partial Unique Index** (ví dụ: `uq_employee_company_code_active`) để nếu nhân viên cũ đã nghỉ việc (bị xóa mềm), Admin hoàn toàn có thể tạo nhân viên mới có cùng mã nhân viên (`employee_code`) mà không bị lỗi ràng buộc database.

### Nhóm 2: Thiết bị & Token đăng nhập (`devices`, `tokens`, `login_approval_requests`)
* **`devices`**: Lưu trữ vân tay thiết bị di động (`device_uid`). Thiết bị mới luôn có trạng thái `pending` và chỉ được dùng để chấm công sau khi Admin duyệt nhằm chống gian lận chấm công hộ.
* **`tokens`**: Quản lý các phiên đăng nhập dựa trên session token tự sinh (không dùng JWT) để đối chiếu thông tin xác thực từ client. Có thể thu hồi (revoke) từ xa khi nhân viên bị mất máy hoặc bị khoá tài khoản.
* **`login_approval_requests`**: Log lưu trữ lịch sử và trạng thái duyệt yêu cầu thiết bị mới từ Admin.

### Nhóm 3: Cấu hình địa điểm GPS & Wifi (`locations`, `location_scopes`, `location_employee_overrides`, `wifis`, `wifi_scopes`)
* **`locations`** & **`wifis`**: Danh mục các vùng GPS và mạng Wifi được phép chấm công. 
* **`location_scopes`** & **`wifi_scopes`**: Gán vị trí/wifi cho Chi nhánh hoặc Phòng ban.
  * *Nguyên lý tối ưu:* Nếu một địa điểm/wifi **không có dòng nào** trong bảng `scopes`, nó mặc định được dùng cho **toàn công ty** (vì bảng cha đã có `company_id`). Nếu có dòng cụ thể, nó sẽ giới hạn phạm vi chỉ phòng ban/chi nhánh đó được dùng. Ràng buộc `chk_location_scope_not_empty` đảm bảo không lưu rác.
* **`location_employee_overrides`**: Dùng cho trường hợp nhân viên đặc cách (ví dụ: đi công tác, làm việc tại nhà) được phép chấm công ở những vị trí đặc thù riêng mà phòng ban của họ không được gán.

### Nhóm 4: Cài đặt Ca làm việc (`shifts`, `shift_assignments`)
* **`shifts`**: Định nghĩa giờ vào/ra, khung giờ checkin/checkout của ca. Nó chỉ quyết định cách thức xác thực (`attendance_method` là GPS, Wifi hoặc cả hai). Địa điểm và Wifi hợp lệ để checkin sẽ được truy vấn động từ chi nhánh/phòng ban của nhân viên đó, tránh việc cấu hình trùng lặp trên mỗi ca.
  * *Tối ưu:* Ràng buộc `chk_shifts_weekdays_values` đảm bảo mảng các thứ trong tuần luôn nằm trong giới hạn cho phép.
  * *Lưu ý nghiệp vụ:* Các cột khung giờ checkout (`checkout_from`, `checkout_to`) đã được thiết lập `NOT NULL` để đảm bảo hệ thống luôn xác định được khoảng thời gian hợp lệ cho punch out.
* **`shift_assignments`**: Gán ca linh hoạt cho toàn công ty, chi nhánh, phòng ban hoặc chỉ định đích danh từng nhân viên. Ràng buộc `chk_shift_assignment_scope` đảm bảo tính toàn vẹn của phạm vi gán (tránh gán nhầm lẫn chồng chéo).

### Nhóm 5: Bản ghi công & Bằng chứng chấm công (`attendance_records`, `attendance_evidences`, `attendance_edit_logs`)
* **`attendance_records`**: Bảng ghi tổng hợp công hàng ngày. Khoá tự nhiên `uq_attendance_employee_shift_date` đảm bảo một nhân viên chỉ có duy nhất 1 dòng ghi công cho mỗi ca trong 1 ngày làm việc.
* **`attendance_evidences`**: Lưu chi tiết bằng chứng của từng lần bấm nút (In/Out) bao gồm: tọa độ GPS thực tế, tên/BSSID Wifi, ảnh chụp, và thiết bị thực hiện.
  * *Tối ưu hóa:* `photo_hash` giúp kiểm tra tính toàn vẹn của ảnh chấm công offline để ngăn chặn việc chỉnh sửa tệp tin giả mạo.

## 3. Kiến Trúc Địa Lý Siêu Tối Ưu (PostGIS)

Thay vì lưu tọa độ thô và tính toán thủ công bằng toán học ở backend, schema này ứng dụng trigger tự động tạo cột dữ liệu địa lý **`geom` (kiểu `geography`)**:
```sql
CREATE TRIGGER trg_locations_geom BEFORE INSERT OR UPDATE OF lat, lng ON locations
FOR EACH ROW EXECUTE FUNCTION set_geography_point();
```
* **Lợi ích:** Backend chỉ cần gửi lên `lat`, `lng` dạng float đơn giản, Database sẽ tự tính toán và điền cột `geom`.
* **Chỉ mục không gian (Spatial Index):** `CREATE INDEX idx_locations_geom ON locations USING GIST(geom)` giúp PostgreSQL tìm kiếm điểm nằm trong bán kính cực kỳ nhanh (phù hợp khi số lượng chi nhánh/văn phòng tăng lên hàng nghìn điểm).

## 4. Bảng So Sánh Chỉ Mục & Tối Ưu Truy Vấn (Index Optimization)

| Mục tiêu truy vấn | SQL Query tương ứng | Index hỗ trợ | Hiệu quả mang lại |
|---|---|---|---|
| **Báo cáo tháng của nhân viên** | `WHERE employee_id = ? AND work_date BETWEEN ? AND ?` | `idx_attendance_records_employee_date` | Tránh quét toàn bộ bảng (Full Table Scan). Truy xuất lịch sử 1 nhân viên chỉ mất vài mili-giây. |
| **Báo cáo tổng hợp phòng ban** | `WHERE company_id = ? AND work_date BETWEEN ? AND ?` | `idx_attendance_records_company_date` | Tính toán báo cáo cho hàng nghìn nhân viên cùng lúc tức thời. |
| **Duyệt chấm công offline** | `WHERE company_id = ? AND approval_status = 'pending'` | `idx_attendance_records_pending_approval` | Admin chỉ cần xem danh sách chờ duyệt, index lọc trực tiếp các dòng `pending` nhanh chóng. |
